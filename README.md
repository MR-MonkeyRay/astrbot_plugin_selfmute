# AstrBot SelfMute 插件

自裁插件 - 移植自 Mirai SelfMute

## 功能介绍

允许群成员使用指令进行自我禁言，支持裸消息和唤醒词两种触发方式。

### 核心特性
- 🎲 随机禁言：不指定时长时，随机禁言 60-3600 秒 × 当日次数
- 🔢 指定时长：可指定禁言秒数（最大 30 天）
- 📅 每日限制：每人每天最多 3 次
- 🛡️ 权限保护：管理员/群主免疫，Bot 需要管理员权限
- 🔒 并发安全：使用异步锁保证数据一致性

## 使用方法

### 基础用法

**默认模式（`use_wake_prefix=false`）：**
```
自裁          # 随机禁言
selfmute      # 随机禁言（英文）
自裁 300      # 禁言 300 秒
/自裁 300     # 也支持唤醒词前缀
```

**唤醒词模式（`use_wake_prefix=true`）：**
```
/自裁 300     # 需要唤醒词前缀
自裁 300      # 不会触发
```

### 禁言规则

**仅群聊可用**，私聊会被拒绝。

1. **随机禁言**：不指定时长时，禁言时长 = 随机值（60-3600秒）× 当日第几次
   - 第 1 次：60-3600 秒
   - 第 2 次：120-7200 秒
   - 第 3 次：180-10800 秒

2. **指定时长**：指定秒数，最大 2,592,000 秒（30 天）

3. **每日限制**：每人每天最多自裁 3 次，跨天自动重置

4. **权限要求**：
   - Bot 必须是群管理员或群主
   - 群管理员和群主对本指令免疫

## 配置选项

### max_daily_count

控制每个用户在单个群组内每日最多可自裁的次数。

- **默认值**: `3`
- **类型**: int

### min_random_seconds

控制随机禁言时，单次随机时长的最小值。

- **默认值**: `60`
- **类型**: int

### max_random_seconds

控制随机禁言时，单次随机时长的最大值。

- **默认值**: `3600`
- **类型**: int

**说明**：实际随机禁言时长 = `随机值 × 当日第几次`。

### use_wake_prefix

控制是否需要唤醒词前缀才能触发指令。

- **默认值**: `false`（支持裸消息触发）
- **类型**: bool

**行为说明：**

| 配置值 | `/自裁 300` | `自裁 300` |
|--------|------------|-----------|
| `false` | ✓ 执行 | ✓ 执行 |
| `true` | ✓ 执行 | ✗ 忽略 |

**注意**：`true` 时，裸消息 `自裁` 不会执行禁言。

## 安装

1. 克隆仓库到 AstrBot 插件目录
2. 重启 AstrBot
3. 确保 Bot 在群内有管理员权限

## 技术实现

- AstrBot Plugin API
- asyncio.Lock 并发控制
- KV 存储持久化
- 薄入口 + 模块化业务层
